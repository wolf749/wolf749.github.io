Pathfinding

In this section I'm going to give an overview of Pathfinding algorithms in video games [and in research].

Pathfinding in games almost exclusively use A*, but navigation fields both discrete and continuous versions (see below) are also gaining traction particularly where pathfinding has to be done for many agents. I've gotten interested in solving pathfinding problems on smooth surfaces, which typically includes solving the geodesic equations.

A* showed up around 1968 and is a graph-based method. Given two nodes on a graph, A* finds the path that minimizes the sum of the costs between the two nodes. Since around 2000, most A* methods in games utilize a navigation mesh, which converts the terrain of the game into a graph data structure. Unity's built-in pathfinding routines are a good example of this.
